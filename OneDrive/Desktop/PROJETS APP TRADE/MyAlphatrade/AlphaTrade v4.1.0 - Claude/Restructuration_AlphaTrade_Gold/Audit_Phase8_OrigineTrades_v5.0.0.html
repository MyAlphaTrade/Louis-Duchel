2. Proposition technique — deux architectures possibles

RECOMMANDÉE

Option B — Ajouter un nom lisible, sans toucher au classement interne

Le champ origin existant ("BOT" / "EXTERNAL_AI" / "MANUAL") reste strictement inchangé — c'est lui que lisent les ~15 endroits du moteur qui décident "est-ce une position à moi ?" (protection, comptage des positions max, stats, rebond…). On ajoute simplement 3 nouveaux champs à côté : origin_name (le vrai nom, ex. "AVA Assistant"), origin_type (le type détaillé du registre), origin_magic (le magic number brut). Le frontend affiche origin_name au lieu de deviner.

Pourquoi : zéro risque sur la boucle de trading en direct — aucun des ~15 filtres qui protègent/comptent/gèrent les positions du bot n'est touché. Seule une information d'affichage est ajoutée.

DÉCONSEILLÉE

Option A — Remplacer origin par un objet structuré partout

Conforme à la demande initiale telle quelle : trade_origin() retourne {origin_name, origin_type, origin_magic} et plus une simple chaîne. Nécessite de réécrire les ~15 filtres origin in ("BOT","ALPHATRADE","ALPHAKARIS") en origin_type == "INTERNAL_BOT", partout dans auto_trade_step, les stats, la protection, le rebond, les clôtures partielles.

Pourquoi déconseillée : ce sont exactement les filtres qui décident quelles positions le bot protège, compte, ferme automatiquement. Une seule condition mal réécrite dans ces ~15 endroits peut faire deux choses graves : le bot cesse de protéger ses propres positions, ou pire, se met à fermer une position manuelle ou externe qui n'est pas la sienne. Risque disproportionné par rapport au bénéfice (le bénéfice — un nom correct affiché — est atteint intégralement par l'Option B, sans ce risque).

2.2 — Nouvelle logique de trade_origin()

def trade_origin(magic, comment, params) -> dict:
    normalized = comment.lower()
    # 1. Magic number exact (priorité la plus haute — le plus fiable)
    # 2. Mots-clés du commentaire
    # 3. Si rien ne correspond : jamais attribué à AVA ni à AlphaTrade par défaut
    for rule in params.get("trade_origins", []):
        if not rule["enabled"]: continue
        if magic in rule["magic_numbers"] or any(kw in normalized for kw in rule["comment_keywords"]):
            return {"origin_name": rule["name"], "origin_type": rule["type"], "origin_magic": magic,
                    "origin": "BOT" if rule["type"] == "INTERNAL_BOT" else "EXTERNAL_AI"}
    if magic == 0:
        return {"origin_name": "Manuel", "origin_type": "MANUAL", "origin_magic": 0, "origin": "MANUAL"}
    return {"origin_name": f"Autre EA ({magic})", "origin_type": "UNKNOWN", "origin_magic": magic, "origin": "EXTERNAL_AI"}

Le dernier champ origin (BOT/EXTERNAL_AI/MANUAL) est calculé automatiquement à partir de origin_type pour que les ~15 filtres existants continuent de fonctionner sans aucune modification (Option B).
